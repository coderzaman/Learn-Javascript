Three Tier Architecture

Presention Layer: * Business Logic Layer

The Presentation Layer is responsible for the user interface and the interaction between the user and the application. It handles all the input and output operations, such as rendering web pages, collecting form data, and presenting information.

This is typically where web pages (HTML, CSS, JavaScript), mobile app interfaces, or desktop applications reside.

Business Logic Layer

The Business Logic Layer is the core of the application, where the actual logic of the application resides. It is responsible for processing user requests, applying business rules, making calculations, and managing workflow.

This layer acts as a mediator between the Presentation Layer and the Data Access Layer, interpreting data and ensuring it is used correctly according to business rules.

Back-end Languages and Frameworks: JavaScript (Node.js), Python (Django, Flask), Java (Spring), Ruby (Rails), PHP (Laravel).

Middleware: REST APIs, GraphQL, etc.

Data Access Layer

The Data Access Layer is responsible for interacting with the database and managing the persistence of data. It handles reading and writing data to a storage medium, such as a database or a file system.

This layer provides an abstraction over raw database operations, ensuring that the business layer doesn't need to interact directly with the database.

Database Management Systems: MySQL, PostgreSQL, MongoDB, SQLite, SQL Server, Oracle.

Object-Relational Mapping (ORM): Sequelize (for Node.js), Hibernate (Java), Django ORM (Python), etc.
